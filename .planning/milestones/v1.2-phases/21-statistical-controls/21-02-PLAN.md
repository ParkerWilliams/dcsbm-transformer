  <action>
Audit `compute_correlation_matrix` in `statistical_controls.py`:

1. **Method identification audit:** The STAT-05 requirement says "Spearman correlation" but the code uses `np.corrcoef` which computes Pearson. Document this discrepancy in test comments. Determine whether this is a bug or an acceptable simplification:
   - If the project's pre-registration or docs explicitly specify Spearman for redundancy, this is a bug requiring a fix (change np.corrcoef to scipy.stats.spearmanr or rank-transform before corrcoef).
   - If Pearson is acceptable for this use case (linear correlation is sufficient for redundancy flagging), document the deviation and test Pearson correctness.
   - Check `docs/pre-registration.md` and any specification for the definitive answer. The 21-CONTEXT.md notes this as a "concrete lead" and says "audit both to determine if Spearman is required per the requirement and fix if needed."
   - If a fix is needed: change `np.corrcoef(data_matrix)` to use `scipy.stats.spearmanr` (rank-based) for measurement mode. For predictive mode, Pearson on AUROC curves is acceptable since AUROC values are already on a bounded scale.

2. **Threshold boundary tests:** Create synthetic metric pairs with controlled correlation:
   - **r ~ 0.89 (below threshold):** Create two metric arrays with Pearson r = 0.89. Use `x = np.random.default_rng(42).normal(0, 1, 200)` and `y = 0.89 * x + sqrt(1 - 0.89^2) * np.random.default_rng(43).normal(0, 1, 200)`. Build fake AnalysisEvents indexing into these arrays. Call `compute_correlation_matrix` in measurement mode. Assert `redundant_pairs` is EMPTY (0.89 is NOT > 0.9).
   - **r ~ 0.91 (above threshold):** Same approach with coefficient 0.91. Assert exactly one redundant pair is returned.
   - **r = 0.90 exactly:** Construct arrays where Pearson r is exactly 0.90 (or as close as floating point allows). Assert NOT flagged (threshold is strict >0.9, not >=0.9). Line 329 of statistical_controls.py uses `r_val > 0.9`.

3. **Pearson correctness check (measurement mode):** Create two metrics with known Pearson correlation (e.g., x and 2*x+1 for r=1.0, x and -x for r=-1.0). Build appropriate AnalysisEvents. Assert the returned matrix values match expected correlations within 1e-6. Assert |r|=1.0 pair IS flagged as redundant.

4. **Predictive mode verification:** Create synthetic metric_arrays and events where the AUROC curves for two metrics are identical. Call `compute_correlation_matrix` in predictive mode. Assert correlation is 1.0 and the pair is flagged redundant. This verifies predictive mode computes AUROC curves correctly before correlating.

5. **Edge cases:** Less than 2 metrics returns no redundant pairs. Less than 3 data points returns identity matrix. Empty events list.

Follow the established audit test pattern: descriptive class per formula aspect, 1-2 line mathematical reasoning comment per assertion.
