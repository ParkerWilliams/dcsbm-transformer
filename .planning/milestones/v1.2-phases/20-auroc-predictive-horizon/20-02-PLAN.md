  <action>
Create `tests/audit/test_event_extraction.py` with tests verifying event extraction correctness. Follow the established audit test pattern.

**class TestExtractEventsOutcomeFiltering:**
Construct synthetic walks producing all 4 outcome types at known positions, verify extract_events only yields FOLLOWED/VIOLATED.

- `test_only_followed_and_violated_events_extracted`: Create a minimal setup:
  - `generated` array with shape (1, 10) containing a jumper vertex at position 2
  - `rule_outcome` array with shape (1, 9) with UNCONSTRAINED at most positions, FOLLOWED at position (resolution_step - 1)
  - `jumper_map` with one jumper (vertex_id=5, r=3, so resolution_step=5, outcome_idx=4)
  - Verify extract_events returns exactly 1 event with outcome=FOLLOWED
  - Then change rule_outcome at that position to VIOLATED, verify outcome=VIOLATED
  - Then change to PENDING, verify no events extracted
  - Then change to UNCONSTRAINED, verify no events extracted

- `test_multiple_encounters_in_single_walk`: Create a walk with 2 jumper encounters at different positions. Set one to FOLLOWED and one to VIOLATED. Verify both events are extracted with correct outcomes.

- `test_resolution_step_calculation`: Verify resolution_step = encounter_step + r. Create event with jumper at step t=3, r=4. Verify event.resolution_step == 7.

- `test_resolution_step_out_of_bounds_skipped`: Create a walk where encounter_step + r would exceed the rule_outcome array length. Verify the encounter is skipped (no event created).

**class TestIsFirstViolation:**
- `test_first_violation_flag_single_violation`: Walk with exactly 1 violation. Verify is_first_violation=True.
- `test_first_violation_flag_multiple_violations`: Walk with 2+ violations. Build failure_index pointing to the first violation's outcome_idx. Only the event whose outcome_idx matches failure_index should have is_first_violation=True.
- `test_followed_event_never_first_violation`: Verify FOLLOWED events always have is_first_violation=False regardless of failure_index.

**class TestContaminationFilter:**
Per CONTEXT.md locked decisions, test asymmetric contamination logic.

- `test_violation_then_nearby_encounter_excluded`: Create events in the same walk: violation at encounter_step=5 with r=3 (resolution_step=8), then another encounter at encounter_step=7 (< 8 = last_violation_end). Second event should be excluded.

- `test_followed_then_nearby_encounter_NOT_excluded`: Create events: FOLLOWED at encounter_step=5 with r=3 (resolution_step=8), then another encounter at encounter_step=7. Since FOLLOWED does NOT set last_violation_end, the second event should NOT be excluded. This is the asymmetric logic.

- `test_violation_with_encounter_outside_window_NOT_excluded`: Violation at encounter_step=5 with r=3 (resolution_step=8), then encounter at encounter_step=9 (>= 8). Second event should NOT be excluded (encounter is after the violation window).

- `test_contamination_audit_statistics`: Verify the audit dict returned by filter_contaminated_events contains correct counts: total_encounters, excluded_encounters, exclusion_rate, and per_r breakdown.

**class TestCrossModuleSeam:**
End-to-end seam test verifying the contract between behavioral.py classification and event_extraction.py.

- `test_behavioral_to_event_extraction_seam`: This test constructs a minimal scenario where we can verify the indexing chain WITHOUT needing a full graph object. Create numpy arrays directly:
  - `generated` with a known jumper vertex at step t
  - `rule_outcome` with VIOLATED at the expected index (t + r - 1)
  - `failure_index` set to (t + r - 1) for the walk
  - `jumper_map` with the jumper's JumperInfo
  Call `extract_events(generated, rule_outcome, failure_index, jumper_map)` and verify:
  - event.encounter_step == t
  - event.resolution_step == t + r
  - event.outcome == RuleOutcome.VIOLATED
  - event.is_first_violation == True (failure_index matches outcome_idx)

- `test_outcome_index_is_resolution_step_minus_one`: Explicitly verify that extract_events looks up rule_outcome at index `resolution_step - 1` (not `resolution_step`). Plant VIOLATED at `resolution_step - 1` and UNCONSTRAINED at `resolution_step` (if in bounds). Verify the event outcome is VIOLATED.

All imports: `extract_events`, `filter_contaminated_events`, `AnalysisEvent` from `src.analysis.event_extraction`, `RuleOutcome` from `src.evaluation.behavioral`, `JumperInfo` from `src.graph.jumpers`.
