2. **test_dcsbm_probability_formula**: For a small graph (n=6, K=2), manually compute expected P values using the formula P_ij = theta_i * theta_j * omega_{z_i, z_j}. 3. **test_dcsbm_no_self_loops**: Verify P[i,i] == 0 for all i. Comment: "Self-loop probability must be zero by construction."

4. **test_dcsbm_degree_correction_preserves_expected_degree**: Verify that per-block theta normalization (sum to block_size) means the expected degree contribution from the uncorrected SBM is preserved. Specifically, for each block b, verify sum(theta[start:end]) == block_size. Comment: "Per-block normalization preserves E[degree] from uncorrected SBM (Karrer & Newman 2011, Section II.A)."

5. **test_dcsbm_block_structure**: Verify that same-block entries use p_in and cross-block entries use p_out in the omega matrix. For vertices i,j in same block: P[i,j] = theta_i * theta_j * p_in. For vertices in different blocks: P[i,j] = theta_i * theta_j * p_out. Comment: "Block assignment drives affinity matrix omega."

6. **test_dcsbm_probability_clipping**: Verify probabilities are clipped to [0, 1]. Create a scenario with high theta values that would push P > 1 without clipping.

7. **test_adjacency_bernoulli_sampling**: For a known P matrix, sample many adjacency matrices and verify empirical edge frequencies match P[i,j] within statistical tolerance. Use n=6, 1000 samples, chi-squared or simple mean comparison. Comment: "Each edge is an independent Bernoulli(P_ij) draw."

8. **test_block_assignment_correctness**: Verify that `np.arange(n) // block_size` correctly assigns vertices to blocks for various n, K values. Include edge case where n=K (block_size=1).

Use pytest, numpy, and scipy.sparse. Use fixed seeds for deterministic results. Each test class should have descriptive names per CONTEXT.md decisions.

Create `tests/audit/test_compliance_rate.py` with the following verification tests for GRAPH-05:

1. **test_compliance_rate_formula_matches_definition**: The mathematical definition of walk compliance rate is: violations / constrained_steps. In the code (`src/training/evaluate.py:evaluate_compliance`), rule_compliance = rule_compliant / total_rule_checks. This is the INVERSE of violation rate: rule_compliance = followed / constrained = 1 - (violations / constrained). Verify this relationship: create a synthetic evaluation scenario with known outcomes. Manually count violations and constrained steps, verify `rule_compliance == (constrained - violations) / constrained == 1 - violation_rate`. Comment: "GRAPH-05: compliance = followed/constrained = 1 - violations/constrained. The code computes the COMPLIANCE rate (fraction correct), not the VIOLATION rate."

2. **test_compliance_rate_all_followed**: When all jumper constraints are met, rule_compliance should be 1.0. Construct sequences where every jumper encounter resolves to the correct target block. Verify rule_compliance == 1.0. Comment: "All constraints satisfied => compliance = 1.0."

3. **test_compliance_rate_all_violated**: When all jumper constraints are violated, rule_compliance should be 0.0. Construct sequences where every jumper encounter resolves to the wrong block. Verify rule_compliance == 0.0. Comment: "All constraints violated => compliance = 0.0."

4. **test_compliance_rate_mixed**: With a known mix (e.g., 3 followed, 2 violated out of 5 constrained steps), verify rule_compliance == 3/5 == 0.6. Comment: "Mixed outcomes: compliance = n_followed / n_constrained."

5. **test_compliance_rate_no_constrained_steps**: When no jumper constraints are active (no jumper encounters or all resolve outside sequence bounds), rule_compliance defaults to 1.0. Verify this matches the code's `if total_rule_checks > 0 else 1.0` default. Comment: "No constrained steps => compliance defaults to 1.0 (vacuously true)."

6. **test_compliance_independent_algebra**: Programmatic algebra check per CONTEXT.md.
