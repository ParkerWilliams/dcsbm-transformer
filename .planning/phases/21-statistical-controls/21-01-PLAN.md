**test_shuffle_permutation.py** — Audit `run_shuffle_control` in `auroc_horizon.py`:

1. **H0 uniformity test:** Generate synthetic data where violated and followed metric values come from the SAME distribution (e.g., N(0,1) for both groups). Create AnalysisEvent objects for 30 violations + 30 controls with sequential walk_idx and resolution_step values that index validly into a metric_array. Run `run_shuffle_control` with n_permutations=1000 independently 100 times (each with a different rng seed). Collect the 100 p-values. Apply scipy.stats.kstest against uniform(0,1). Assert KS p-value > 0.01 (H0: p-values are uniform should NOT be rejected). This confirms no false signal under the null.

2. **Metric array immutability:** Create a metric_array, copy it with `.copy()`, run `run_shuffle_control`, then assert `np.array_equal(original, metric_array_copy)` — the function must not mutate the metric values during permutation. Also verify the function permutes labels (via the index mask approach), not the metric values themselves.

3. **Group size preservation per-permutation:** This is an internal property. To verify, create a small standalone test that replicates the permutation logic: combine n_viol + n_ctrl events, call `rng.permutation(n_total)`, take first n_viol as violation mask. Assert exactly n_viol True values in the mask. Run 50 such iterations to confirm.

4. **Signal detection (positive control):** Generate data where violations are drawn from N(5, 1) and controls from N(0, 1) (clear separation). Run `run_shuffle_control` with n_permutations=1000. Assert p_value < 0.05 — the signal is real and should not be explained by shuffling.

**test_holm_bonferroni.py** — Audit `holm_bonferroni` in `statistical_controls.py`:

1. **Textbook worked example:** Use the classic 5-hypothesis example from Holm (1979). Input p-values: [0.01, 0.04, 0.03, 0.005, 0.5]. The sorted order is [0.005, 0.01, 0.03, 0.04, 0.5]. Multiplied by [5, 4, 3, 2, 1] gives [0.025, 0.04, 0.09, 0.08, 0.5]. After monotonicity enforcement: [0.025, 0.04, 0.09, 0.09, 0.5]. At alpha=0.05, reject first two (0.025 <= 0.05, 0.04 <= 0.05). Verify adjusted p-values and reject flags match exactly.

2. **Monotonicity enforcement test:** Construct p-values where raw adjustment would break monotonicity. Example: p_values = [0.01, 0.001, 0.5]. Sorted: [0.001, 0.01, 0.5]. Multiplied by [3, 2, 1]: [0.003, 0.02, 0.5]. No monotonicity issue here. Instead use: p_values = [0.01, 0.04, 0.02]. Sorted: [0.01, 0.02, 0.04]. Multiplied by [3, 2, 1]: [0.03, 0.04, 0.04]. After monotonicity: [0.03, 0.04, 0.04]. Now try a case that DOES break: p_values = [0.04, 0.01, 0.03]. Sorted: [0.01, 0.03, 0.04]. Multiplied by [3, 2, 1]: [0.03, 0.06, 0.04]. Without monotonicity: p3 < p2 which violates step-down ordering. After enforcement: [0.03, 0.06, 0.06]. Verify the code produces this.

3. **Edge cases:** Empty input returns empty arrays. Single p-value returns itself (clipped at 1.0). All identical p-values. All p-values = 0.0. All p-values = 1.0.

4. **Equivalence check with (m - k + 1) formula:** The code uses 0-based `(m - i)`. The textbook uses 1-based `(m - k + 1)`. For m=5 and k=1..5: (m-k+1) = [5,4,3,2,1]. For i=0..4: (m-i) = [5,4,3,2,1]. Assert these are identical for all m from 1 to 10.

Follow the established audit test pattern: descriptive class per formula aspect, 1-2 line mathematical reasoning comment per assertion.

**test_cohens_d.py** — Audit `cohens_d` in `statistical_controls.py`:

1. **Hand-calculation verification:** Use group1 = [2, 4, 6] (mean=4, var=4, n=3) and group2 = [1, 2, 3] (mean=2, var=1, n=3). Pooled std = sqrt(((3-1)*4 + (3-1)*1) / (3+3-2)) = sqrt((8+2)/4) = sqrt(2.5) ~ 1.5811. Cohen's d = (4-2) / sqrt(2.5) = 2/1.5811 ~ 1.2649. Assert `cohens_d(group1, group2)` matches within 1e-4.

2. **Sign convention:** group1 > group2 should give positive d. group1 < group2 should give negative d. Assert sign is correct.

3. **Known exact case:** group1 = [0, 2] (mean=1, var=2), group2 = [0, 0] (mean=0, var=0). Pooled std = sqrt(((2-1)*2 + (2-1)*0) / (2+2-2)) = sqrt(2/2) = 1.0. d = (1-0)/1.0 = 1.0. Assert exact match.

4. **NaN guard edge cases:**
   - Single-element groups (n<2): `cohens_d(np.array([1.0]), np.array([2.0, 3.0]))` returns NaN.
   - Identical-value groups: `cohens_d(np.array([5.0, 5.0, 5.0]), np.array([5.0, 5.0, 5.0]))` — pooled_std < 1e-12, returns NaN.
   - Normal groups return finite values (use group1 = [1,2,3,4,5], group2 = [2,3,4,5,6]).

5. **Independent pooled std calculation:** Recompute pooled_std using a completely separate implementation (manual loop or different numpy call path) and compare against what `cohens_d` implicitly uses (back-calculate from the output).

Follow the established audit test pattern from Phases 18-20.
