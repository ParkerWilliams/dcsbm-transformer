  <action>
Create `src/reporting/templates/audit_report.html` — a Jinja2 template for the self-contained audit report.

**Template receives these variables from the Python generator:**
- `katex_css`: str — full KaTeX CSS content (inlined)
- `katex_js`: str — full KaTeX JS content (inlined)
- `categories`: list[str] — category names in order
- `entries_by_category`: dict[str, list[dict]] — entries grouped by category
- `summary`: dict — per-category verdict counts: `{category: {correct: N, fixed: N, concern: N}}`
- `total_correct`: int, `total_fixed`: int, `total_concern`: int
- `generation_date`: str

**Layout structure (per user decisions):**

1. **Sticky left sidebar TOC** (~240px wide, position:fixed, overflow-y:auto):
   - Report title at top
   - Category links that scroll to sections (anchor links)
   - Under each category, list individual requirement IDs as sub-links
   - Highlight active section on scroll (optional, at Claude's discretion)

2. **Main content area** (margin-left to clear sidebar):

   a. **Header**: "v1.2 Mathematical Audit Report" with generation date

   b. **Summary dashboard**: Table with columns: Category, Correct, Fixed, Concern, Total
      - One row per category with counts
      - Final row with totals
      - Color-code the cells: green for correct, amber for fixed, red for concern (background tint)

   c. **Category sections** (one per category, with `id` for anchor links):
      - Category heading (h2)
      - Cards for each requirement entry

   d. **Requirement cards** (bordered box with subtle shadow):
      - Header row: requirement ID + title on left, verdict badge on right
      - Verdict badge: pill-shaped span with color (green "Correct", amber "Fixed", red "Concern")
      - LaTeX formula block: a div with class "formula-block" that KaTeX will render
      - Code location: monospace text showing file:line
      - Fix description: shown inline below verdict if verdict is "fixed", otherwise hidden

3. **KaTeX rendering script** at bottom of body:
   - After page load, find all elements with class "formula-block"
   - For each, call `katex.render(element.dataset.latex, element, {displayMode: true, throwOnError: false})`
   - The LaTeX string is stored in a `data-latex` attribute on the element (Jinja2 escapes it)

**Styling (academic aesthetic per Phase 9 established patterns):**
- Body font: Georgia, "Times New Roman", serif
- Sidebar background: #f8f9fa, border-right: 1px solid #ddd
- Card: border: 1px solid #e0e0e0, border-radius: 6px, padding: 1.5em, margin-bottom: 1.5em, background: #fff
- Verdict badges: display:inline-block, padding: 0.2em 0.8em, border-radius: 12px, font-size: 0.85em, font-weight: bold
  - Correct: background #d4edda, color #155724
  - Fixed: background #fff3cd, color #856404
  - Concern: background #f8d7da, color #721c24
- Code location: font-family monospace, color #555, font-size 0.9em
- Formula block: margin 1em 0, min-height 2em
- Responsive: on narrow screens (<768px), sidebar collapses to top bar

**Important:** The LaTeX content must be stored in `data-latex` attributes on elements (not rendered server-side), because KaTeX renders client-side in the browser. Use Jinja2's `|e` (escape) filter for the data attribute value to handle special characters. Wrap in a CDATA-style approach if needed: store the raw LaTeX in a `<script type="text/latex">` tag inside each card's formula div, then have the rendering script read from those tags.

**Self-contained:** All CSS is in `<style>` tags. KaTeX CSS and JS are in `<style>` and `<script>` tags respectively. No external references.
